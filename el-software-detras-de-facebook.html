el software detrás de Facebook

Date dom 13 febrero 2011 Tags cloud / mysql / facebook / php / hadoop

Siempre he tenido curiosidad de Facebook. Te conectas y se carga todo en apenas segundos. A día de hoy, Facebook tiene 500 millones de usuarios activos. ¿Cómo pueden hacerlo para que vaya tan rápido? Buscando he encontrado que, para soportar esto, en Facebook han tenido que romper con la manera tradicional de servir contenido y evolucionar hacia nuevas soluciones. A parte de Oracle RAC (en esta presentación, transparencia 11), el uso de soluciones de software libre es apabullante.

Los números que maneja actualmente son:

  • 570.000 millones de páginas vistas al mes.
  • Tiene más fotos que el resto de sítios de fotos combinados (incluido Flickr).
  • Se suben 3.000 millones de fotos todos los meses y sirve más de 1.2 millones de estas por segundo.
  • Todo esto lo manejan más de 30.000 servidores.

¿Y cual es el software que ayuda a manejar todo esto?

Aunque aún mantiene parte de su plataforma LAMP, ha ido evolucionando creando un compilador propio de PHP, linux con kernels parcheados por ellos mismos y llevando lógica de MySQL al propio servidor web. Además de esto, se ayuda de: Hadoop / Hive

Cómo no, Hive sobre Hadoop está teniendo cada vez más peso en la arquitectura de Facebook. Al igual que Cassandra, son del proyecto Apache y son de código abierto. Se ejecutan sobre Java y se utilizan para el proceso de grandes cantidades de datos. Hive se ejecuta sobre Hadoop, por lo que puede distribuir el trabajo en tantos servidores como tenga el cluster. En este blog ya hablamos sobre cómo instalar Hadoop en una máquina Ubuntu.

Haystack

Facebook utiliza un servidor http que utiliza Haystack para guardar y servir las fotos. Es un servidor adaptado y optimizado para este propósito. Haystack ha sido desarrollado por Facebook, para migrar desde NFS.

Memcached

Memcached es un sistema distribuido para cachear la información que fluye entre el servidor de base de datos y el servidor web. Éste ha sido modificado y adaptado por Facebook, siempre buscando las optimizaciones que necesitaban.

HipHop

HipHopconvierte PHP en C++, que luego es compilado para obtener mejor rendimiento que en PHP nativo.

BigPipe

BigPipees un servidor de páginas web que utiliza Facebook para servir cada página en secciones llamadas "pagelets" para optimizar el rendimiento. De esta manera se puede obtener en paralelo varias partes de la misma página, aumentando la velocidad de carga. Si te fijas mientras se carga la página que estás solicitando, las diferentes secciones se cargan en tiempos distintos.

Cassandra

Se utiliza para las búsquedas del Inbox. Cassandraes una base de datos NoSQL que fue desarrollada por Facebook y liberada posteriormente. Actualmente forma parte del proyecto Apache.

Como vemos, en Facebook se utiliza un compendio de herramientas de software libre bastante punteras (no me refiero a LAMP) las cuales, o las han desarrollado ellos, o las utilizan, mejoran y amplían de manera activa.
